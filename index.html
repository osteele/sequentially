Sequentially: Adverbs for JavaScript (preview version)

header | announcement | download: compressed (<2K min gz) | source

Sequentially is a library of temporal and frequency adverbs for JavaScript. It provides methods to queue a function for deferred or repeated execution, and to control the rate or total count at which a function can be called.

I use this to write client-side programs that do things like periodically ask an application server to recreate a resource if it's not available from an asset server, but throttle the requests and eventually back off and then give up if it looks like the server is too busy or down.

Sequentially also contains a partial port Haskell's MVars, that can each be used to coordinate data flow among asynchronously-scheduled functions.

Functionally, Sequentially combines data flow programming with simple process scheduling. Stylistically, Sequentially can be used in adverbial programming, a paradigm that complements procedural (verb-base), object-oriented (noun-based), and functional (gerundive) programming. Theoretically, Sequentially uses closures to embed (some of) the π-calculus in JavaScript. Alternatively, it's the CPS conversion of the categorial dual of generators. Or maybe it's the Chu space dual of Functional JavaScript. Or, it's nothing more than a few combinators for frequency, iteration, and time.

This is a preview release. It hasn't been thoroughly tested, and I plan to make some major changes to the API as I continue to experiment how best to use this in my own programs.
